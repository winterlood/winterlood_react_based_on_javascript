# 자료형과 형 변환

안녕하세요 winterlood입니다.

이번시간에는 자료형과 그런 자료형을 변경하는 기능인 형 변환에 대해 알아보도록 하겠습니다.

## 01. 자바스크립트의 자료형

자료형이라는 것은 값의 성질에 따라 분류한 것 입니다.

이를테면 25라는 값은 숫자형으로 분류할 수 있으며, "25"는 문자열 형으로 분류할 수 있습니다.

자바스크립트는 값을 크게는 2개, 작게는 8개의 자료형으로 분류합니다.

![](https://user-images.githubusercontent.com/46296754/140057660-39fa4948-f7a4-49c3-ae15-2e922939e4ab.png)

우리는 이 8개의 자료형 중 `Primitive Data Type` 5개 즉 원시타입 자료형에 대해서만 알아보겠습니다

나머지 비 원시타입 자료형인 객체, 배열과 함수는 뒤에서 배울것이니 지금은 몰라도 괜찮습니다.

## 02. 원시 타입

원시 타입이라는 것은 무슨 의미일까요? 원시타입이라는 말은 내장형 타입 또는 기본형타입 이라고도 부를 수 있는데요

![](https://user-images.githubusercontent.com/46296754/140242652-b8a86322-304f-4586-af1c-08caf7513fdb.png)

원시타입의 가장 대표적인 자료형인 숫자형을 보시면, 하나의 순간에는 하나의 값만 가질 수 있습니다.

하나의 순간이라는 표현이 조금 어렵게 느껴지실 수 있는데요, 이 뜻은 프로그램이 실행되는 도중에, 숫자형 값을 담고 있는 변수는 그 값이 바뀔수는 있지만 언제나 딱 하나의 값만 가지고 있다고 생각하시면 됩니다.

이렇게 하나의 고정된 저장공간을 갖는 데이터를 원시 타입이라고 부를 수 있습니다.

비 원시타입은 값을 하나만 가지지 않는 경우의 자료형을 의미합니다. 이를테면 `array`라는 배열처럼 하나의 순간에 `[1,2,3,4]`이렇게 4가지의 숫자형 값을 가지고 있는 것 처럼 한번에 여러개의 값을 가질 수 있습니다.

이번 강의에서는 비 원시타입 자료형은 다루지 않고, 원시타입 자료형들에 대해서만 다룰 예정입니다.

원시타입 자료형을 하나 하나씩 실습을 통해 살펴보는 시간을 갖도록 하겠습니다.

### 2-1. 숫자형 number

첫번째로 알아볼 원시타입 자료형은 대부분의 언어에서 가장 기본이 되는 자료형인 숫자형입니다.

```javascript
let age = 25;
let tall = 175.9;
```

그냥 숫자는 다 숫자형입니다 소수던 실수던 다 숫자형 타입이라고 생각하시면 됩니다.

```javascript
let age = 25;
let tall = 175.9;

console.log(age * 20); // 500
```

숫자형 타입의 값들은 우리가 생각하는 사칙연산을 수행할 수 있습니다.

숫자형 타입은 이 정도로만 알아보도록 하겠습니다.

### 2-2. 문자형 string

두번째 원시 타입은 문자열 타입 string입니다.

이름과 같은 값을 저장할 때 사용할 수 있겠네요 한번 해보도록 하겠습니다

```javascript
let name = "winterlood";
```

`name`이라는 변수를 만들고, `name`에 "winterlood"라는 문자열 값을 할당해주었습니다.

string type은 이렇게 따옴표로 문자들을 감싸서 표현할 수 있습니다.

지금은 쌍 따옴표를 사용했는데요 string type에 사용되는 따옴표는 쌍 따옴표를 포함해서 총 세개입니다.

첫번째는 쌍따옴표이고, 두번째는 작은 따옴표인데요 뭐 큰따옴표나 차이는 없습니다.

세번째 따옴표가 조금 특별합니다. 이 따옴표를 백틱이라고 하는데요 우리 키보드의 숫자 1번 왼쪽에 있는 물결모양을 쉬프트 클릭 없이 누르면 나오는 그 친구가 백틱입니다.

```javascript
let name = `winterlood`;
```

이렇게 백틱으로 감싸도 문자열을 사용할 수 있는데, 이 백틱을 이용하면 문자열 속에 값을 넣을 수 있습니다.

```javascript
let tall = 175.9;
let name = `winterlood's tall : ${tall}`;
```

숫자형 값을 가지는 변수 `tall`을 생성하고 이것을 백틱으로 감싼 문자열 속에 `${}` 를 사용해서 넣을 수 있습니다.

이러한 방법을 `템플릿 리터럴` 이라고 부릅니다. 이 `템플릿 리터럴`은 향후에 아주 유용하게 사용되므로 꼭 익혀두시는걸 추천드립니다.

### 2-3. 불린형 boolean

세번째 원시타입은 불린형 입니다.

이 불린형은 참이나 거짓만을 저장하는 자료형입니다. 뭐 예로들자면 스위치가 꺼졌냐 켜졌냐, 그 값이 참이냐 거짓이냐 등을 저장할 대 유용하게 사용할 수 있습니다.

```javascript
let isSwitchOff = true;

let isTrue = false;
```

### 2-4. null

네번째 원시타입은 `null`입니다. `null`은 아무것도 없다 라는 것을 지칭하는 값이자 자료형인데요, 보통은 이 `null`은 다른 언어에서는 그냥 값으로만 활용되는데 자바스크립트에서는 특별하게 자료형으로 분류됩니다.

```javascript
let trash = null;
console.log(trash);
```

이 `null` 또한 값이기 때문에 변수에 저장해서 사용할 수 있습니다.

다른 언어를 사용해보신 분들은 여기서 한가지 착각하실 수 있는데요

```javascript
let variable;
```

이렇게 변수를 생성하고 아무런 초기화도 하지 않으면 그러니까 아무 값도 담지 않으면 기본적으로 `null`을 가지는거라고 생각하실 수 있는데요

```javascript
let variable;
console.log(variable); // undefiend
```

자바스크립트에서는 이런 상황에서 `null`값이 아닌 `undefiend`라는 값을 가지게 됩니다.

### 2-5. undefined

마지막 원시타입 자료형인 `undefined`입니다.

우리가 앞으로 가장 자주 보게 될 친구입니다. 그리고 자주 에러를 발생시키는 원인이 되는 가장 무섭기도 한 친구입니다.

방금 실습해 보았던 것 처럼 변수를 선언하고 초기화 하지 않으면 그 변수는 `undefined`이라는 자료형이자 값을 할당받게 됩니다.

```javascript
let variable;
console.log(variable); // undefiend
```

## 03. 형 변환

### 3-1. 형 변환 이란?

```javascript
let numberA = 12;
let numberB = 2;
```

이 `numberA`와 `numberB`를 곱셈한 값을 `console.log`로 출력하면 어떤 값이 나오나요?

```javascript
let numberA = 12;
let numberB = 2;

console.log(numberA * numberB); // 24
```

24라는 숫자와 숫자의 곱셈 결과가 나오게 됩니다. 숫자와 숫자를 곱했으니까 당연히 우리가 아는 사칙연산의 곱셈이 적용된 사례입니다.

```javascript
let numberA = 12;
let numberB = "2";

// console.log(numberA * numberB);
```

잠시 `console`은 지워두고 `numberB`를 숫자형 2가 아니라 문자열 2로 바꿔보겠습니다.

만약 `numberA * numberB`를 수행하면 결과값이 어떻게 될까요?

문자열과 숫자를 곱하는 연산은 우리가 배웠던 수학에는 존재하지 않는 연산입니다.

결과가 어떻게 되는지 `console.log`를 통해 알아보도록 하겠습니다.

```javascript
let numberA = 12;
let numberB = "2";

console.log(numberA * numberB); //24
```

24라는 결과값이 나왔는데요, 이것은 2를 숫자로 인지하여 계산한 결과일 것 입니다.

문자열과 숫자를 곱하려 했더니, 문자열 2를 숫자 2로 자동으로 자료형을 변환하여 결과를 나타내주었습니다.

자바스크립트 코드를 읽고 해석하여 실행하는 자바스크립트 엔진은 이렇게, 다른 자료형간의 연산을 수행할 때 적절하게 자료형을 자동으로 변환하여 결과를 돌려줍니다.

이렇게 값을 유지하면서 자료형을 변환시키는 기능을 `Casting(형 변환)`이라고 부릅니다.

그리고 우리가 의도하지 않은 이런 형 변환이 일어나는 것은 묵시적으로 그러니까 암묵적으로 알아서 해준다 라는 뜻을 담아 `묵시적 형변환` 이라고 부릅니다.

### 3-2. 명시적 형 변환

이번에는 문자열과 숫자의 곱셈이 아닌, 덧셈을 해보도록 하겠습니다.

```javascript
let numberA = 12;
let numberB = "2";

console.log(numberA + numberB); // 122
```

`122`라는 결과가 나왔는데요, 이는 숫자 12가 문자열로 변환되어 문자열을 이어붙인 결과입니다.

문자열과 숫자의 곱셈은 문자열을 숫자로 바꿔 수행했으나, 문자열과 숫자의 덧셈은 숫자를 문자열로 바꿔 문자열을 이어붙입니다.

만약 이때 `numberA`를 문자열로 변환하지 않고 `numberB`를 숫자형으로 변환하여 결과를 `14`를 만들고 싶다면 아래와 같이 하면 됩니다.

```javascript
let numberA = 12;
let numberB = "2";

console.log(numberA + parseInt(numberB)); //14
```

`parseInt` 내장함수는 문자열을 받아 숫자로 리턴해줍니다. 우리가 이 `parseInt`에 문자열 2를 넣었으니 숫자형 2를 리턴하였고, 12와 2의 덧셈인 14가 정상적으로 나오는것을 볼 수 있습니다.

이렇게 프로그래머가 내장함수 등을 이용하여 값을 유지하며 자료형을 의도적으로 변환시키는 행위는 `명시적 형변환`이라고 부릅니다.

이번 강의에서는 자료형과 캐스팅(형 변환)에 대해서 알아 보았습니다. 이 자료형과 형 변환은 익숙해지지 않는다면 자바스크립트를 사용하는 그 마지막 날까지 우리를 괴롭힐 것이 분명하므로 익숙해지시기를 추천드립니다.

다음 강의에서는 자바스크립트의 덧셈이나 뺄셈같은 연산을 명시할 수 있는 `연산자`에 대해 이야기 해보도록 하겠습니다.
