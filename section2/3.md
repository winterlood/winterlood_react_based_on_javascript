# 자료형과 형 변환

## 01. 자바스크립트의 자료형

자료형이라는 것은 값의 성질에 따라 분류한 것 입니다.

이를테면 25라는 값은 숫자형으로 분류할 수 있으며, "25"는 문자열 형으로, 함수는 함수형이라고 볼 수 있습니다.

자바스크립트는 값을 크게는 2개, 작게는 8개의 자료형으로 분류합니다.

![](https://user-images.githubusercontent.com/46296754/140057660-39fa4948-f7a4-49c3-ae15-2e922939e4ab.png)

우리는 이 8개의 자료형 중 `Primitive Data Type` 5개 즉 원시타입 자료형에 대해서만 알아보겠습니다

나머지 비 원시타입 자료형인 객체, 배열과 함수는 뒤에서 배울것이니 지금은 몰라도 괜찮습니다.

## 02. 원시 타입

원시타입 자료형부터 시작하도록 하겠습니다.

원시타입이라는 것은, 내장형 또는 기본형이라고도 부를 수 있는데요

하나의 고정된 저장공간을 갖는 데이터를 원시 타입이라고 부를 수 있습니다.

예를들어 숫자는 한번에 하나의 숫자만 지정할 수 있습니다. 이건 당연한거죠?

그러나 비 원시타입의 배열같은 경우 [1,2,3]이렇게 3개의 저장공간을 가집니다. 이렇게 저장공간을 하나 가지는 데이터 타입을 원시타입, 그렇지 않은 녀석들을 비원시 타입이라고 부릅니다.

이제 하나씩 살펴보도록 하겠습니다.

### 2-1. 숫자형 number

첫번째로 알아볼 원시타입 자료형은 대부분의 언어에서 가장 기본이 되는 자료형인 숫자형입니다.

```javascript
let age = 25;
let tall = 175.9;
```

그냥 숫자는 다 숫자형입니다 소수던 실수던 다 숫자형 타입이라고 생각하시면 됩니다.

### 2-2. 문자형 string

두번째 원시 타입은 문자열 타입 string입니다.

```javascript
let name = "winterlood";
```

string type은 이렇게 따옴표로 문자들을 감싸서 표현할 수 있습니다.

지금은 쌍 따옴표를 사용했는데요 string type에 사용되는 따옴표는 쌍 따옴표를 포함해서 총 세개입니다.

첫번째는 쌍따옴표이고, 두번째는 작은 따옴표인데요 뭐 큰따옴표나 차이는 없습니다.

세번째 따옴표가 조금 특별합니다. 이 따옴표를 백틱이라고 하는데요 우리 키보드의 숫자 1번 왼쪽에 있는 물결모양을 쉬프트 클릭 없이 누르면 나오는 그 친구가 백틱입니다.

```javascript
let name = `winterlood`;
```

이렇게 백틱으로 감싸도 문자열을 사용할 수 있는데, 이 백틱을 이용하면 문자열 속에 값을 넣을 수 있습니다.

```javascript
let tall = 175.9;
let name = `winterlood's tall : ${tall}`;
```

숫자형 값을 가지는 변수 `tall`을 생성하고 이것을 백틱으로 감싼 문자열 속에 `${}` 를 사용해서 넣을 수 있습니다.

이러한 방법을 `템플릿 리터럴` 이라고 부릅니다. 이 `템플릿 리터럴`은 향후에 아주 유용하게 사용되므로 꼭 익혀두시는걸 추천드립니다.

### 2-3. 불린형 boolean

세번째 원시타입은 불린형 입니다.

이 불린형은 참이나 거짓만을 저장하는 자료형입니다. 뭐 예로들자면 스위치가 꺼졌냐 켜졌냐, 그 값이 참이냐 거짓이냐 등을 저장할 대 유용하게 사용할 수 있습니다.

```javascript
let isSwitchOff = true;

let isTrue = false;
```

### 2-4. null

네번째 원시타입은 `null`입니다. `null`은 아무것도 없다 라는 것을 지칭하는 값이자 자료형인데요, 보통은 이 `null`은 다른 언어에서는 그냥 값으로만 활용되는데 자바스크립트에서는 특별하게 자료형으로 분류됩니다.

```javascript
let trash = null;
console.log(trash);
```

이 `null` 또한 값이기 때문에 변수에 저장해서 사용할 수 있습니다.

다른 언어를 사용해보신 분들은 여기서 한가지 착각하실 수 있는데요

```javascript
let variable;
```

이렇게 변수를 생성하고 아무런 초기화도 하지 않으면 그러니까 아무 값도 담지 않으면 기본적으로 `null`을 가지는거라고 생각하실 수 있는데요

```javascript
let variable;
console.log(variable); // undefiend
```

자바스크립트에서는 이런 상황에서 `null`값이 아닌 `undefiend`라는 값을 가지게 됩니다.

### 2-5. undefined

마지막 원시타입 자료형인 `undefined`입니다.

우리가 앞으로 가장 자주 보게 될 친구입니다. 그리고 자주 에러를 발생시키는 원인이 되는 가장 무섭기도 한 친구입니다.

방금 실습해 보았던 것 처럼 변수를 선언하고 초기화 하지 않으면 그 변수는 `undefined`이라는 자료형이자 값을 할당받게 됩니다.

```javascript
let variable;
console.log(variable); // undefiend
```

## 03. 형 변환

### 3-1. 형 변환 이란?

형 변환이란, 특정 연산을 수행하기 위해서 적절하게 자료형을 변경하는 것을 의미합니다.

아래의 코드에서는 문자열 "5" 곱하기 문자열 "2"를 수행했습니다.
문자열 끼리의 곱셈은 자바스크립트에 정의되어있지 않습니다.

따라서 오류가 발생해도 이상하지 않을 상황이지만, 자바스크립트는 이 상황에서 자동으로 연산에 맞는 자료형으로 형 변환을 진행합니다.

즉 문자열 5와 문자열 2가 숫자형 5와 2로 변환되어 계산되는 것 입니다.

```javacsript
console.log("5" * "2"); // 10
```

이러한 것을 형변환이라고 부릅니다.

### 3-2. 문자형으로 변환

문자형으로의 타입 변환은 두가지 함수를 사용할 수 있습니다.

> 1. String() 함수
> 2. value.toString()함수

두가지 함수 모두 동일한 값을 반환하지만, null이나 undefined에는 String()함수만 사용할 수 있습니다.

아래의 예제를 통해 문자형으로 변환을 연습해 봅시다.

```javascript
let value = true;
console.log(typeof value); // boolean

value = String(value); // 변수 value엔 문자열 "true"가 저장됩니다.
console.log(typeof value); // string

let value2 = true;
console.log(typeof value2); // boolean

value2 = value2.toString(); // 변수 value2 문자열 "true"가 저장됩니다.
console.log(typeof value2); // string
```

### 3-3. 숫자형으로 변환

숫자형으로의 변환 또한 어렵지 않게 수행할 수 있습니다.

숫자형으로 변환하는 메소드or함수는 대표적으로 아래 4가지가 있습니다.

> 1. Number(arg)
> 2. parseInt(arg)
> 3. parseFloat(arg)

하나씩 살펴보면서 그 차이를 느껴보도록 하겠습니다.

#### 2-3-1. Number(arg)

이 메서드는 무조건 숫자로 이루어진 것만 숫자로 반환합니다.

아래 예제에서 처럼 "2014년"은 한글 문자가 포함되었기에, 숫자로 반환하지 못하고 NaN을 반환하고 있는 것을 보실 수 있습니다.

```javascript
let str1 = "2014";
let str2 = "2014년";
console.log(Number(str1)); // 2014
console.log(Number(str2)); // NaN
```

#### 2-3-2. parseInt(arg)

숫자와 문자가 섞여있어도 숫자만 인식하여 돌려줍니다 (정수)

단, 변환할 수 없는 문자에 도달할 때 까지, 문자열에서 가능한 많은 문자를 읽어 숫자로 변환하는 방식을 따르기 때문에, 아래의 코드 str2의 경우에는 2014까지 숫자로 변환하나, str3의 경우에는 시작과 동시에 문자를 만나므로 변환에 실패하는 모습을 볼 수 있습니다.

```javascript
let str1 = "2014";
let str2 = "2014년";
let str3 = "올해는 2021년";
console.log(parseInt(str1)); // 2014
console.log(parseInt(str2)); // 2014
console.log(parseInt(str3)); // NaN
```

#### 2-3-3. parseFloat(arg)

숫자에는 정수만 포함되어있는건 아닙니다 소수 포함되어있죠 parseFloat는 정수로 반환하는 parseInt와는 다르게 값을 소수로 반환합니다
아래의 코드 예제는 모든 라인이 3.14를 반환합니다.

> 정확히 표현하자면 부동소수점 소수로 값을 반환합니다.

```javascript
console.log(parseFloat(3.14));
console.log(parseFloat("3.14"));
console.log(parseFloat("314e-2"));
console.log(parseFloat("0.0314E+2"));
console.log(parseFloat("3.14와 숫자가 아닌 문자들"));
```
